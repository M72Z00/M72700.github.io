Autonomous Water Filtering Cart [C++]

The Autonomous Water Filtering Cart is an innovative solution designed to address the persistent challenge of poor water quality and the limited access to clean filtered water. As part of my senior design project during my Computer Engineering major, I worked collaboratively with a team of four to develop this advanced cart, integrating a variety of technologies to ensure effective and reliable water filtration.

Project Overview:

  • Objective: To create an autonomous system that improves water quality and provides a practical solution for accessing clean filtered water in areas with limited resources.
  • Technologies Used:
    • Arduino Software: The heart of the cart's control system, managing the various sensors and components through an Arduino microcontroller.
    • 7-Filter System: A sophisticated filtration setup designed to purify water effectively, ensuring it meets clean water standards.
    • GPS Tracking Device: Provides location data to monitor and navigate the cart's position, ensuring it operates efficiently and reaches designated areas.
    • Line Sensing/Tracking: Allows the cart to follow predefined paths or routes, improving its operational efficiency and accuracy.
    • Ultrasonic Sensors: Used to measure water levels and ensure proper operation of the filtering system by detecting the presence of water.
    • Solar Powered Battery: Powers the cart sustainably using solar energy, enhancing its mobility and reducing reliance on external power sources.

Significance of the Project:

The Autonomous Water Filtering Cart represents a significant advancement in addressing water quality issues through automation and smart technology. This project showcases the integration of various engineering concepts and practical applications:

  • Automation and Control: The system autonomously manages water filtration processes, minimizing the need for manual intervention and ensuring consistent water quality.
  • Real-time Monitoring: Sensors provide continuous feedback on water levels and filter performance, allowing for timely adjustments and improvements.
  • Sustainability: The use of solar power emphasizes the project's commitment to sustainable and eco-friendly solutions.
  • Practical Impact: By improving access to clean water, the cart contributes to better public health and supports communities in need.

This project, developed as part of a team during my senior year, demonstrates a comprehensive application of engineering principles and problem-solving skills, reflecting my ability to contribute effectively to complex, real-world challenges.

Pac-Man [Scratch Project]

This Pac-Man game showcases fundamental programming concepts through a classic arcade-style experience. In this single-level game, I’ve implemented several key features:

  • Ghost AI: The four ghosts are programmed with simple artificial intelligence to track Pac-Man's movements, providing a dynamic challenge. When Pac-Man consumes a power-up, the ghosts switch behavior, retreating to their spawn point at the maze's center to avoid being captured.
  • Collision Detection: The game accurately detects collisions between Pac-Man, the ghosts, and the circle-shaped food pellets, updating the score and displaying the eaten items in real-time.
  • Event Handling and Sound Integration: The game responds to various player actions, such as moving Pac-Man and eating food, with corresponding sound effects, enhancing the user experience.

This project demonstrates an understanding of game loops, conditionals, and event-driven programming, all essential skills for developing interactive applications.

Flying Space Cat [Scratch Project]

This Flying Space Cat game was developed as part of a coding club project, where elementary students were introduced to key programming concepts through a fun and interactive experience. The game features:

  • Sprite Movement: Players control Space Cat using the up and down arrow keys, allowing them to navigate through the galaxy. The movement mechanics teach students how to manipulate sprites and respond to user input.
  • Scrolling Background: The game includes a continuously scrolling background, creating the illusion of Space Cat flying through space. This element introduces students to the concept of background looping and animation.
  • Collision Detection: As Space Cat collects diamonds, the game checks for collisions between the cat and the gems. This helps students understand how to detect interactions between different game elements.
  • Game-Over Condition: The game ends if the player misses 3 diamonds, providing a clear example of how to implement game-over logic and manage player lives.

This project effectively combines basic game development techniques, helping students build a strong foundation in programming while engaging them with a creative and enjoyable challenge.

Guessing Number 1-100 Game [Java]

This Java-based guessing game was developed during my time as a student at California State University, Sacramento. The project was created to demonstrate fundamental programming concepts through an engaging console application. The game includes:

  • Random Number Generation: The game generates a random number between 1 and 100, teaching the use of randomization in programming and how to handle dynamic game elements.
  • User Input Handling: Players input their guesses, and the game provides feedback on whether the guesses are too high, too low, or correct. This introduces concepts of user input processing and conditional statements.
  • Guess Counting: The game tracks the number of attempts made by the player to guess the correct number. This feature demonstrates how to implement and update counters within a program.
  • Replay Option: After a correct guess, the game offers an option to play again. This teaches how to use loops and method calls to create a continuous game experience.
  • Error Handling: The game includes checks for invalid input, guiding users to enter integers within the specified range. This helps in understanding how to handle and validate user input effectively.

This project effectively combines essential programming techniques and error handling, providing a hands-on learning experience in Java programming through a simple yet interactive game.

Pac-Man [Python]

This Pac-Man game, developed using Python and Pygame, is a classic arcade-style game that features a single-level maze for Pac-Man to navigate. The game combines the beloved elements of the original Pac-Man with modern programming practices, including customizable ghost AI and collision detection.

Game Features:

  • Pac-Man: Controlled by the player, Pac-Man is represented as a yellow circle. The player's goal is to navigate Pac-Man through the maze to collect pellets while avoiding ghosts.
  • Ghosts: The game features a ghost with basic AI, programmed to either chase Pac-Man or flee from it. The ghost’s behavior is determined by its AI type, making each encounter unique.
  • Maze Layout: The maze is composed of walls and pellets. Walls are randomly sized and placed, while pellets are scattered throughout the maze. The maze's layout challenges the player to navigate carefully to avoid collisions.
  • Gameplay Mechanics:
    • Movement: Pac-Man moves in four directions (up, down, left, right) based on player input.
    • Collision Detection: The game checks for collisions between Pac-Man, walls, and pellets. Collisions with walls prevent Pac-Man from passing through, while collisions with pellets increase the player’s score.
    • Ghost Interaction: If Pac-Man collides with the ghost, the player loses a life. The game ends when all lives are lost.
    • Level Completion: The level is considered complete when all pellets have been collected, prompting a win message.
  • User Interface:
    • Start Screen: Displays a message prompting the player to press any key to start the game.
    • In-Game HUD: Shows the current score and number of lives remaining.
    • End Screens: Includes messages for game over and level completion, with options to restart or quit the game.
  • Technical Details:
    • Collision Handling: Uses Pygame's Rect objects for accurate collision detection between Pac-Man, ghosts, walls, and pellets.
    • AI Mechanics: The ghost uses simple vector mathematics to move towards or away from Pac-Man, depending on its AI type.
    • Frame Rate Control: The game runs at 30 frames per second to ensure smooth gameplay.

This game demonstrates fundamental game development concepts such as game loops, event handling, collision detection, and simple AI. It offers a nostalgic experience while showcasing your skills in Python and Pygame.
